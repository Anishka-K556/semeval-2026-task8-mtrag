# Task A — Conversational Passage Retrieval

**SemEval-2026 Task 8 (MTRAG-UN) — Document Retrieval Subtask**

Retrieves the top-10 most relevant passages for each query across four domains:
ClapNQ, IBM Cloud, FiQA, and Govt.

---

## Pipeline

```
Multi-turn conversation
        |
        v  Stage 1 — Query Rewriting
Gemma-2-2B-IT  (4-bit NF4, few-shot, greedy decoding)
        |
        v
Standalone rewritten query
        |
        +----------> BM25 (lexical)              top-150  |
        +----------> BAAI/bge-base-en-v1.5        top-150  +--> RRF (k=60) --> top-100
        +----------> intfloat/e5-base-v2           top-150  |
        |
        v  Stage 2 — Ensemble Reranking
        BAAI/bge-reranker-large             x 0.40
        mixedbread-ai/mxbai-rerank-large-v1 x 0.35
        cross-encoder/ms-marco-MiniLM-L-12  x 0.25
        |
        v
Top-10 passages with scores
```

Reranker weights were tuned on the dev set using nDCG@5.

```bash
pip install faiss-gpu
```

Gemma query rewriting requires a GPU with ~3 GB+ VRAM (4-bit quantisation is
used by default). The pipeline falls back to CPU but is very slow without a GPU.

## Configuration

All paths and hyperparameters are controlled from `config.json` at the repo
root. The only value you need to set is `base_path`:

```json
{
  "base_path": "/path/to/your/SemEval/data",
  ...
}
```

Everything else (corpus paths, query paths, embedding paths, output paths) is
derived automatically. Expected data layout under `base_path`:

```
SemEval/

│   ├── cloud.jsonl
│   ├── fiqa.jsonl
```

## Step-by-step usage

### Step 0 — Generate corpus embeddings (run once per domain)

Encodes each domain corpus with both BGE and E5 and saves them as `.pkl` files.
Only needs to be run once — use `--skip-existing` on subsequent runs.

```bash
python Task-A/preprocessing/generate_embeddings.py --domain clapnq
python Task-A/preprocessing/generate_embeddings.py --domain ibmcloud
python Task-A/preprocessing/generate_embeddings.py --domain fiqa
python Task-A/preprocessing/generate_embeddings.py --domain govt

# Skip domains whose .pkl files already exist
python Task-A/preprocessing/generate_embeddings.py --domain clapnq --skip-existing
```

### Step 1 — Rewrite queries with Gemma

Converts multi-turn last-turn queries into standalone queries by resolving
coreferences and implicit context.

```bash
# Single domain
python Task-A/query_rewriting/rewrite.py --domain clapnq

# All domains at once
python Task-A/query_rewriting/rewrite.py --domain all
```

Reads `<domain>_lastturn.jsonl`, writes `<domain>_rewrite.jsonl` to the same
queries directory.

### Step 2 — Run the retrieval pipeline

**Dev mode** — runs the full pipeline and prints Recall and nDCG metrics at the end:

```bash
# Single domain
python Task-A/retrieval/pipeline.py --domain clapnq

# All domains (models are shared across domains — much faster)
python Task-A/retrieval/pipeline.py --domain all
```

Each domain writes `results/<domain>_taskA_results.jsonl`.

**Test mode** — skips qrels loading and evaluation, outputs submission-ready JSONL:

```bash
# All domains
python Task-A/retrieval/pipeline.py --domain all --test

# Point at a custom test query file for a single domain
python Task-A/retrieval/pipeline.py --domain clapnq --test \
    --query-path /path/to/clapnq_test_queries.jsonl
```

### Step 3 — Combine into a single submission file

```bash
python Task-A/combine_results.py
```

Merges the four per-domain result files into `final_submission_taskA.jsonl`
with exactly 10 contexts per record.

### Step 4 — Evaluate on the dev set

```bash
# Single domain
python Task-A/evaluate.py \
    --results results/clapnq_taskA_results.jsonl \
    --qrels   /path/to/qrels/clapnq/dev.tsv

# All domains with macro average
python Task-A/evaluate.py --submission final_submission_taskA.jsonl
```

## Key hyperparameters

| Parameter | Value | Config key |
|---|---|---|
| Candidates per retriever | 150 | `retrieval.top_retrieval` |
| After RRF fusion | 100 | `retrieval.top_fusion` |
| RRF k | 60 | `retrieval.rrf_k` |
| Final top-K passages | 10 | `retrieval.final_top_k` |
| Reranker batch size | 16 | `reranking.batch_size` |
| BAAI/bge-reranker-large weight | 0.40 | `reranking.weights.bge_reranker_large` |
| mixedbread-ai/mxbai-rerank-large-v1 weight | 0.35 | `reranking.weights.mxbai_rerank_large` |
| cross-encoder/ms-marco-MiniLM-L-12-v2 weight | 0.25 | `reranking.weights.ms_marco_minilm` |
